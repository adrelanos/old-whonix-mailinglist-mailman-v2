  Not sure what your aim is.

  But I'm using guix parallel to my normal Linux system. Guix uses it's
own file-hierarchy, not interfering with the "host" system. This only
point to take care is how to start services in the guix "sphere".

  As far as I understand these packages, they contain exactly what is
described in <https://nixos.org/wiki/Installing_Nix_on_Debian> [1]. So
they may save some half an hour of setup time. But then you rely on
these packages to been updated as frequently as nix itself. …

  … which I'm afraid of: It looks like are no longer providing these
packages :-( The last one I spotted was
<https://nixos.org/releases/nix/nix-1.11.2/> [2] (dated February 2016),
while 1.11.6 is current.

  BTW: Setting up guix is equally easy:
https://www.gnu.org/software/guix/manual/html_node/Installation.html [3]
