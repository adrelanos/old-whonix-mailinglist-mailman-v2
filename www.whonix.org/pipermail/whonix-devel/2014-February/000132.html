M. Edward (Ed) Borasky:
> I'm looking at the KVM instructions now. I think there's one or
> two unnecessary steps.

These steps aren't required for the test version anymore at all. Since
.qcow2 images are now available.

> And I don't think you don't need to convert VMDK -> VDI -> QCOW2.
> qemu-img will convert from VMDK to QCOW2 directly.

qemu-img convert "vmdk_file" -O RAW "vdi_file"

qemu-img version 1.6.1 fails with:
qemu-img: 'image' uses a vmdk feature which is not supported by this
qemu version: VMDK version 3

This is a known issue in qemu:
https://bugs.launchpad.net/qemu/+bug/1253465

> First of all, Virtual Machine Manager / qemu-kvm *can* work with
> VMDK files directly.

Are you sure about that? Perhaps with VMDK version 2. With VMDK
version 3 as well?

(VirtualBox beginning with the version that is in Debian Jessie uses
VMDK version 3 by default, i.e. Whonix 7 VirtualBox .ova's include
VMDK version 3. Not sure about the new test images which have been
build on the older Debian Wheezy.)

Guestmount doesn't support mounting VMDK version 3 either, because
that internally uses qemu-img as well. Therefore I would wonder if KVM
is capable of working with VMDK version 3 directly. Happy to hear
otherwise.

Fortunately, for KVM the VMDK / image conversion stuff won't be
required anymore since .qcow2 images are now available.
